# CURRENT FOCUS

This document describes the current engineering focus of the project. It is intentionally short and should always reflect the active development priorities.

---

## Current milestone

Build a stable and explainable training platform with reliable data collection, deterministic readiness calculations, and production-ready infrastructure.

---

## Current priorities

### 1. Platform stability

- Improve backend reliability.
- Strengthen monitoring and observability.
- Expand automated testing.
- Simplify deployment.

---

### 2. iOS application

- Improve HealthKit synchronization.
- Refine dashboard UX.
- Increase sync reliability.
- Continue architecture cleanup.

---

### 3. Readiness Engine

- Improve deterministic readiness calculations.
- Increase explainability.
- Prepare for future adaptive models.
- Keep algorithms transparent.

---

### 4. Data pipeline

- Improve HealthKit ingestion.
- Improve Strava integration.
- Ensure data consistency.
- Preserve historical data.

---

## Avoid

Unless explicitly requested, avoid spending time on:

- Large-scale refactoring.
- Experimental AI features.
- New infrastructure.
- Premature optimization.
- Cosmetic changes without product value.

---

## Working style

Prefer:

- Small pull requests.
- Incremental improvements.
- Existing patterns.
- Simple implementations.
- Clear documentation.

Always optimize for maintainability over cleverness.
